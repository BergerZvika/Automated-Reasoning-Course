PySMT Lab

Python API for SMT solving — formulas, solvers, and models

Python / PySMT
PySMT is a Python library providing a unified API to SMT solvers (Z3, MathSAT, CVC4…).  ·  Build formulas with Symbol, And, Or, Plus, Equals, etc.  ·  Solve them with Solver or is_sat.  ·  Retrieve models with get_value / get_model.  ·  Install: pip install pysmt then pysmt-install --z3
Code Examples
pysmt-hello_world.py Python 

      

    

    

      

        Hello World
        
Letter-value puzzle

        
Assign integer values 1–10 to the letters in "hello" and "world" such that their sums are equal and both equal 25.

      

      

        

          
Symbol(name, type)

          
Creates a typed variable. Use INT, REAL, or BOOL from pysmt.typing.

        

        

          
LE(a, b) / GE(a, b)

          
Less-than-or-equal / Greater-than-or-equal constraints between two terms.

        

        

          
And(...) / Plus(...)

          
And takes any number of formulas. Plus sums a list of terms. Both accept generators.

        

        

          
Equals(a, b)

          
Equality constraint between two terms. Different from Python's == which compares objects.

        

        

          
Solver(logic="QF_LIA")

          
Context-manager solver. QF_LIA = Quantifier-Free Linear Integer Arithmetic.

        

        

          
solver.get_value(symbol)

          
Returns the model value as a PySMT constant. Use .is_true() for booleans, or compare with TRUE().

        Boolean SAT
        
Basic propositional solving

        
Create Boolean symbols, build a formula with And/Or/Not, solve it, and inspect model values.

      

      

        

          
Symbol(name, BOOL)

          
Boolean variable. Import BOOL from pysmt.shortcuts directly.

        

        

          
Solver(name="z3")

          
Instantiate a specific backend solver by name. Requires that solver to be installed via pysmt-install.

        

        

          
solver.add_assertion(f)

          
Assert a formula to the solver. Multiple calls are implicitly conjoined.

        

        

          
solver.check_sat()

          
Returns True if SAT, False if UNSAT.

        

        

          
value.is_true() / value == TRUE()

          
Model values are PySMT objects, NOT Python booleans. Always use .is_true() or compare with TRUE().

        LIA / LIRA
        
Integer and Real arithmetic

        
Mix integer and real variables in the same formula. PySMT keeps them typed; Z3 handles mixed-arithmetic via LIRA logic.

      

      

        

          
Symbol("x", INT) / Symbol("y", REAL)

          
Import INT and REAL from pysmt.typing. A symbol's type is fixed at creation.

        

        

          
Int(n) / Real(r)

          
Typed numeric constants. Always use these instead of bare Python literals to avoid type mismatches.

        

        

          
Plus(x, Int(5))

          
Arithmetic addition. Also available: Minus, Times, Div.

        

        

          
y < Real(5.0)

          
PySMT overloads Python comparison operators on symbols, producing PySMT formula objects (not Python booleans).

        

        

          
solver.get_model()

          
Returns a Model object. Print it directly to see all variable assignments.

        Portfolio
        
Running multiple solvers in parallel

        
A Portfolio solver races several backends and returns the first answer. Useful when different solvers are faster on different inputs.

      

      

        

          
Portfolio(["z3","msat"], logic=…)

          
Accepts a list of solver names and an explicit logic. Spawns all solvers simultaneously and returns the first result.

        

        

          
from pysmt.logics import QF_BOOL

          
Logic objects from pysmt.logics are more precise than string names and enable solver selection optimisation.

        

        

          
is_sat(formula)

          
Shorthand from pysmt.shortcuts — creates a solver, checks, and disposes. Good for one-shot checks.

        Quantifiers
        
Universal quantification with ForAll

        
Check whether ∀x. x+1 < x is satisfiable. This is a contradiction — no integer satisfies it — so the result is UNSAT.

      

      

        

          
ForAll([vars], formula)

          
Universal quantifier. The first argument is a list of bound variables. Produces a quantified formula.

        

        

          
Exists([vars], formula)

          
Existential quantifier — not used here but works the same way. For satisfiability, ∃x.φ(x) is checked directly.

        

        

          
LT(a, b)

          
Strict less-than. Equivalent to a < b using operator overloading.

        

        

          
Quantified logic

          
Quantified formulas go beyond QF (quantifier-free) logics. Z3 supports them via AUFLIA, LIA, etc. Not all solvers handle quantifiers.
